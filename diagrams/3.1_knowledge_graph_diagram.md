# 3.1 The Innovation Knowledge Graph Diagram

This diagram visualizes the topological structure of the Innovation Knowledge Graph built on Neo4j. It illustrates how discrete data points (like user quotes and observations) serve as foundational nodes that are explicitly tethered via strongly typed edges to synthesized insights and downstream product features, creating a mathematically queryable chain of custody.

```mermaid
graph TD
    %% Entities
    subgraph Raw Data
        Quote["(:UserQuote)\n'The checkout is confusing'"]
        Observation["(:Observation)\nUser abandoned cart"]
    end
    
    subgraph Synthesis
        Insight["(:Insight)\nFriction in payment flow"]
        POV["(:POV)\nGen-Z users need transparent pricing"]
        HMW["(:HMW)\nHow might we clarify fees?"]
    end
    
    subgraph Execution
        Feature["(:Feature)\nTransparent Fee Toggle"]
    end
    
    %% Relationships
    Quote -->|"[:SYNTHESIZED_FROM]"| Insight
    Observation -->|"[:SYNTHESIZED_FROM]"| Insight
    Insight -->|"[:VALIDATES]"| POV
    POV -->|"[:GENERATES]"| HMW
    HMW -->|"[:INFORMS]"| Feature
    
    %% Styling
    classDef raw fill:#e3f2fd,stroke:#1565c0,stroke-width:2px;
    classDef synth fill:#fff3e0,stroke:#e65100,stroke-width:2px;
    classDef exec fill:#e8f5e9,stroke:#2e7d32,stroke-width:2px;
    
    class Quote,Observation raw;
    class Insight,POV,HMW synth;
    class Feature exec;
```
