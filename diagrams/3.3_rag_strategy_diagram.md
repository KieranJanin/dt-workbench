# 3.3 Retrieval-Augmented Generation (RAG) Strategy Diagram

This diagram maps out the advanced RAG execution layer that overrides default LLM behaviors. It shows the parallel hybrid search bridging Pinecone and Neo4j, the dynamic prompt templating, and the rigid Pydantic output parser that legally mandates the LLM to mathematically cite upstream graph nodes, ensuring a hallucination-free output.

```mermaid
graph TD
    %% User Trigger
    User["Innovation Team Prompt:\n'Synthesize features for POV'"]
    
    %% Orchestration
    subgraph API Middleware Orchestrator
        Hybrid["Parallel Hybrid Search"]
        
        subgraph Databases
            Pinecone[("Pinecone\n(Semantic Historical Data)")]
            Neo4j[("Neo4j Graph\n(Relational Context & Citations)")]
        end
        
        Template["Dynamic Few-Shot Prompt Templating"]
    end
    
    %% LLM Execution
    subgraph AI Reasoning Layer
        LLM{"Primary LLM Inference"}
        Parser{"Rigid Output Parser\n(Pydantic Schema)"}
    end
    
    %% Presentation
    UI["Next.js Presentation Layer\n(Interactive Micro-Links)"]
    Reject["AI Mediation Layer\n(Reject Hallucinations)"]
    
    %% Flow
    User --> Hybrid
    Hybrid --> Pinecone
    Hybrid --> Neo4j
    Pinecone & Neo4j -->|"Collated Context Packet"| Template
    
    Template --> LLM
    LLM -->|"Forces Upstream Citations\n(e.g., Quote_ID)"| Parser
    
    Parser -->|Invalid / Hallucination| Reject
    Parser -->|Validated Payload| UI
    
    %% Styling
    classDef user fill:#e3f2fd,stroke:#1565c0,stroke-width:2px;
    classDef orch fill:#fff3e0,stroke:#e65100,stroke-width:2px;
    classDef ai fill:#ede7f6,stroke:#4527a0,stroke-width:2px;
    
    class User,UI user;
    class Hybrid,Template,Parser orch;
    class LLM,Reject ai;
    class Pinecone,Neo4j fill:#f3e5f5,stroke:#6a1b9a;
```
